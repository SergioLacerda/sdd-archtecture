## 🎯 Workflow (ADR-007 + ADR-008)

### For Agent (me):

```
1. Design Phase (Day 1)
   ├─ Create FEATURE_DESIGN_*.md
   ├─ git checkout -b wip/v3.1-beta.1-designs
   ├─ git add FEATURE_DESIGN_*.md
   ├─ git commit -m "design: v3.1-beta.1 core features"
   ├─ git push origin wip/v3.1-beta.1-designs
   └─ Wait for architect approval (async)

2. Spec Phase (Day 2)
   ├─ Create COMPONENT_*.md
   ├─ Write test cases (not implementation!)
   ├─ git add COMPONENT_*.md
   ├─ git commit -m "spec: v3.1-beta.1 technical specs"
   ├─ git push origin wip/v3.1-beta.1-*
   └─ Mark ready for implementation

3. Implementation Phase (Day 3-4)
   ├─ Implement Layer 1-4 (code)
   ├─ Run tests (all 111+ pass)
   ├─ Update docs
   ├─ git add . && git commit -m "feat: Implement v3.1-beta.1"
   ├─ Create PR on GitHub
   ├─ Title: [REVIEW] v3.1-beta.1 Core Features
   └─ Wait for architect review + merge
```

### For Architect (you):

```
1. Design Review (Day 1-2)
   ├─ Read FEATURE_DESIGN_*.md files
   ├─ Verify: Matches architectural decisions?
   ├─ Verify: Scope is clear?
   ├─ Signal: ✅ APPROVED or ⚠️ REVISE

2. PR Review (Day 4)
   ├─ Check: Code matches spec?
   ├─ Check: Tests pass (111/111)?
   ├─ Check: Docs updated?
   ├─ Signal: ✅ APPROVED (merge) or ⚠️ REVISE

3. Merge (Day 4)
   ├─ Only architect can commit to main
   ├─ Agent cannot force-push
   ├─ Backup critical files before merge
   └─ Verify main CI/CD passes
```
