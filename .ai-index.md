## 📋 Critical Rules You Must Know

### The 16 Mandatory Rules

File: `EXECUTION/docs/ia/CANONICAL/rules/ia-rules.md`

**Subset of most critical:**

1. **Ports Mandatory** — Never import infrastructure directly
   - Use ports/interfaces, not concrete implementations
   - Example: `from src.domain.ports import StoragePort` ✅
   - NOT: `from src.infrastructure.storage import DatabaseAdapter` ❌

2. **Thread Isolation** — Only modify YOUR thread
   - Check `EXECUTION/docs/ia/custom/[PROJECT]/development/execution-state/`
   - If someone is working on it, wait or coordinate

3. **Tests During Implementation** — TDD always
   - Write test first, then code
   - Never "implement first, test later"

4. **No Implicit State** — Always verify before acting
   - Check files exist before modifying
   - Verify paths are correct

5. **No Direct Infrastructure Imports** — Use ports
   - See rule #1 above

6-16. [See full file for complete rules](./EXECUTION/docs/ia/CANONICAL/rules/ia-rules.md)

## 🔑 Key Concepts You Must Understand

### Ports & Adapters Pattern (ADR-003)
Your code should be organized like this:

```
src/[domain]/
├── ports/                    (interfaces)
│   ├── storage_port.py       (what storage should do)
│   └── notification_port.py  (what notifications should do)
├── use_cases/               (business logic)
│   └── create_thing.py       (uses ports, not implementations)
└── adapters/                (implementations)
    ├── postgres_storage.py   (implements StoragePort)
    └── email_notification.py (implements NotificationPort)
```

**Rule:** Import ports from `src.domain.ports`, never from `src.infrastructure`

### Definition of Done (45+ Criteria)
File: `EXECUTION/docs/ia/CANONICAL/specifications/definition-of-done.md`

Your feature isn't done until you check all 45+ items:
- ✅ Tests passing (100% coverage for new code)
- ✅ Documentation updated
- ✅ No rule violations
- ✅ Architecture validated
- ✅ (40+ more items)

### Thread Isolation
Each person/agent works on ONE task at a time.

- Check: `EXECUTION/docs/ia/custom/[YOUR_PROJECT]/development/execution-state/_current.md`
- If someone else is working on your task: Wait or coordinate
- After you're done: Update checkpoint with what you did

### Checkpointing
After finishing work, document it:

```markdown
# Checkpoint Entry [TIMESTAMP @ AGENT_NAME]

**TASK:** What did you implement?
**DECISIONS:** What choices did you make?
**QUESTIONS:** What's unclear?
**RISKS:** What could go wrong?
**STATUS:** Ready for PR
```

This helps the next person know what you did.
